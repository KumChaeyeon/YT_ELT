# 01. 프로젝트 개요

- 본 프로젝트는 **데이터 엔지니어링의 전체 흐름을 이해하고, 이를 실제로 구현하는 것**을 목표로 한다.
- 단순히 개별 기술(API, Python 등)을 학습하는 것뿐만 아니라,  
  **데이터 수집 → 저장 → 처리 → 테스트 → 배포까지 이어지는 전체 데이터 파이프라인 구축 과정**을 중심으로 진행한다.
- 이를 통해 데이터 엔지니어링이 단순한 데이터 처리 작업이 아니라,  
  **다양한 기술이 결합된 시스템 설계 과정**임을 이해한다.

---

# 02. 데이터 엔지니어링 전체 흐름

본 과정은 다음과 같은 단계로 구성되며, 이는 실제 데이터 파이프라인의 흐름과 동일하다.
```
Data Extraction → Data Loading & Transformation → Testing → CI/CD
```

- 외부 API를 통해 데이터를 수집하고,
- 데이터를 저장 및 가공한 뒤,
- 테스트를 통해 검증하고,
- 자동화 및 배포를 수행하는 구조이다.

→ 이를 통해 데이터 엔지니어링은  
**단일 기술이 아닌 여러 단계가 유기적으로 연결된 과정**임을 이해할 수 있다.

---

## 1) Data Extraction (데이터 수집 단계)

- 외부 데이터 소스로부터 데이터를 수집하는 과정
- 본 프로젝트에서는 **YouTube API**를 활용하여 특정 채널(MrBeast)의 데이터를 수집

### 주요 학습 내용
- API 구조 이해
- API Key 발급 및 인증 처리
- Python을 이용한 HTTP 요청 수행
- JSON 데이터 파싱

→ 외부 시스템과의 데이터 연동 방법 학습  
→ 여러 단계의 요청이 필요한 구조를 통해 데이터 수집의 복잡성 이해

---

## 2) Data Loading & Transformation (데이터 적재 및 변환)

- 수집된 데이터는 그대로 사용할 수 없기 때문에 저장 및 가공 과정이 필요

### 주요 개념
- ETL / ELT 구조
- 데이터 정제 및 구조화
- 데이터베이스(PostgreSQL) 적재
- Python 기반 데이터 처리

→ 핵심:  
**데이터를 단순 저장하는 것이 아니라, 분석 가능한 형태로 변환하는 과정**

---

## 3) Functional & E2E Testing (기능 및 통합 테스트)

- 데이터 파이프라인의 정상 동작 여부를 검증하는 단계

### 테스트 유형
- Unit Test (단위 테스트)
- Integration Test (통합 테스트)
- End-to-End Test (전체 흐름 테스트)

→ 데이터 파이프라인 역시  
**소프트웨어 시스템처럼 검증 가능한 구조로 설계되어야 함**

---

## 4) Data Quality Testing (데이터 품질 검증)

- 데이터의 정확성과 신뢰성을 확보하기 위한 단계

### 주요 검증 요소
- 결측값 (Missing Data)
- 중복 데이터 (Duplicates)
- 형식 오류 (Incorrect Formats)

→ 데이터 품질 관리 또한  
**데이터 엔지니어의 핵심 역할 중 하나**

---

## 5) CI/CD (지속적 통합 및 배포)

- 코드 자동화 및 배포를 위한 단계

### 주요 개념
- GitHub 기반 코드 관리
- 자동화된 테스트 실행
- CI/CD 파이프라인 구성

→ 데이터 파이프라인은 단순 실행 코드가 아니라  
**자동화된 시스템으로 운영되어야 함**

---

## 6) 전체 구조에 대한 종합적 이해

본 프로젝트를 통해 데이터 엔지니어링은 다음과 같은 흐름을 포함하는 통합 시스템임을 확인하였다.
```
데이터 수집 → 저장 → 가공 → 검증 → 자동화
```

- 각 단계는 독립적으로 동작하지만, 전체적으로 연결되어 하나의 파이프라인을 구성한다.
- 따라서 특정 기술 하나보다,  
  **전체 흐름 속에서 각 기술의 역할을 이해하는 것이 중요하다.**

---

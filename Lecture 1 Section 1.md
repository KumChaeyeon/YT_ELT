# 01. 프로젝트 개요

- 본 프로젝트는 데이터 엔지니어링의 전체 흐름을 이해하고, 이를 실제로 구현하는 것이 목표
- 단순히 개별 기술(API, Python 등)을 학습하는 것 뿐만 아니라,  
**데이터 수집부터 저장, 처리, 테스트, 배포까지 이어지는 전체 데이터 파이프라인을 구축하는 과정**을 중심으로 진행
- 데이터 엔지니어링이 단순한 데이터 처리 작업이 아니라, 다양한 기술이 결합된 시스템 설계 과정임을 확인

# 02. 데이터 엔지니어링 전체 흐름

- 본 과정은 다음과 같은 단계로 구성되며, 이는 실제 데이터 파이프라인의 흐름과 동일
```
Data Extraction → Data Loading & Transformation → Testing → CI/CD
```
    
- 먼저, 외부 API를 통해 데이터를 수집하고, 이를 저장 및 가공한 뒤, 테스트를 통해 검증하고, 마지막으로 자동화 및 배포를 수행하는 구조
- 이 흐름을 통해 데이터 엔지니어링은 **단일 기술이 아니라 여러 단계가 유기적으로 연결된 과정임을 이해**

## 1) Data Extraction (데이터 수집 단계)

- 외부 데이터 소스로부터 데이터를 수집하는 과정
    - 본 프로젝트에서는 YouTube API를 활용하여 특정 채널(MrBeast)의 데이터를 수집
- 다음과 같은 작업이 포함
    - API 구조 이해
    - API Key 발급 및 인증 처리
    - Python을 이용한 HTTP 요청 수행
    - JSON 데이터 파싱
- 이를 통해 외부 시스템과의 데이터 연동 방법을 학습
- 또한, 단일 요청이 아닌, 여러 단계의 요청을 통해 데이터를 수집해야 한다는 점에서 데이터 수집 과정의 복잡성을 확인

## 2) Data Loading & Transformation (데이터 적재 및 변환)

- 수집된 데이터는 바로 활용할 수 있는 형태가 아니므로, 이를 저장하고 가공하는 과정이 필요
- 다음과 같은 개념 이해 필요
    - ETL/ELT 구조
    - 데이터 정제 및 구조화
    - 데이터베이스(PostgreSQL) 적재
    - Python을 활용한 데이터 처리
- 특히, 데이터를 단순 저장하는 것이 아니라, **분석 가능한 형태로 변환하는 과정이 핵심**임을 확인

## 3) Functional & E2E Testing (기능 및 통합 테스트)

- 데이터 파이프라인이 정상적으로 동작하는지 검증하기 위한 테스트 단계가 필요
- 다음과 같은 테스트 개념을 학습
    - Unit Test (개별 기능 테스트)
    - Integration Test (시스템 간 연결 테스트)
    - End-to-End Test (전체 흐름 테스트)
- 이를 통해 데이터 파이프라인 또한 소프트웨어 시스템과 동일하게 **검증 가능한 구조로 설계되어야 함**을 이해

## 4) Data Quality Testing (데이터 품질 검증)

- 데이터 엔지니어링에서 중요한 요소 중 하나는 데이터 품질
- 단순히 데이터를 수집하는 것만으로는 충분하지 않으며, 데이터가 정확하고 신뢰할 수 있는지 검증해야 함
- 다음과 같은 문제를 고려
    - 결측값 (missing data)
    - 중복 데이터 (duplicates)
    - 형식 오류 (incorrect formats)
- 이를 통해 데이터 품질 관리 또한 데이터 엔지니어의 중요한 역할임을 확인

## 5) CI/CD (지속적 통합 및 배포)

- 코드의 자동화 및 배포
- 다음과 같은 개념을 학습
    - Github 기반 코드 관리
    - 자동화된 테스트 실행
    - CI/CD 파이프라인 구성
- 이를 통해 데이터 파이프라인 역시 단순 실행 코드가 아니라, **자동화된 시스템으로 운영되어야 함**을 이해

## 6) 전체 구조에 대한 종합적 이해

- 본 프로젝트를 통해 데이터 엔지니어링은 단순히 데이터 처리 기술이 아니라, 다음과 같은 흐름을 포함하는 통합적인 시스템임을 확인
```
데이터 수집 -> 저장 -> 가공 -> 검증 -> 자동화
```
    
- 각 단계는 독립적인 기능을 수행하지만, 전체적으로 연결되어 하나의 파이프라인을 구성
- 따라서 특정 기술 하나를 학습하는 것보다, **전체 흐름 속에서 각 기술의 역할을 이해하는 것이 중요**하다고 판단

# 03. 결론

- 본 Section에서는 데이터 엔지니어링 프로젝트의 전체 구조를 파악하고, 각 단계에서 수행해야 할 작업과 사용되는 기술을 정리
    - 이를 통해 이후 진행되는 데이터 추출(API), 데이터 저장(PostgreSQL), 자동화(Docker, Airflow) 등의 학습이 어떤 맥락에서 이루어지는지 이해할 수 있었음
    - 특히, 본 과정은 단순한 기술 학습이 아니라, 실제 데이터 엔지니어링 시스템을 구축하기 위한 단계별 설계 과정으로 구성되어 있음
